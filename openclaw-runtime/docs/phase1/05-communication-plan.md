# OpenClaw Runtime 阶段1沟通计划

## 1. 每日站会安排

- 时间：每日 10:00（15 分钟）
- 参与：Runtime、Router、QA、Tech Lead
- 模板：
  - 昨日完成
  - 今日计划
  - 阻塞项
  - 风险变化

## 2. 进度汇报机制

- 日报：提交到 `docs/phase1/daily/`（可选）
- 里程碑汇报：M1/M2/M3/M4 完成当日同步
- 指标同步：成功率、P95 延迟、失败任务数

## 3. 问题升级流程

### 3.1 升级等级
- P0：系统不可用/数据不一致扩大。
- P1：核心路径不稳定但可降级。
- P2：功能缺陷或性能波动。

### 3.2 响应时效
- P0：10 分钟内拉群 + 30 分钟内给临时策略。
- P1：2 小时内给修复方案。
- P2：24 小时内纳入迭代。

## 4. 文档更新计划

- 架构文档：每个里程碑后更新。
- API 契约：字段变更实时同步。
- 回滚手册：M2 后必须成文，M4 前演练通过。

## 5. 协作边界（大脑-手脚）

- 大脑（competition_router）负责：
  - 复杂度分层
  - 任务分发建议
  - 融合决策

- 手脚（openclaw-runtime）负责：
  - 依赖调度执行
  - skill 调用与容错
  - 标准化执行回传

## 6. 会议节奏建议

- 周一：计划与风险评审
- 周三：中期技术评审（关注架构偏差）
- 周五：里程碑验收 + 下周调整
